# 架构分析任务

## 任务概述
对目标系统进行全面的架构五视图分析，包括逻辑架构、开发架构、部署架构、运行架构和数据架构。

## 输入
- 技术文档或代码库
- 系统架构描述
- 配置文件

## 输出
- 架构五视图分析报告
- 架构质量评估
- 改进建议

## 执行步骤

### 1. 逻辑架构分析
- 识别系统的主要组件和模块
- 分析组件间的依赖关系
- 识别架构模式和设计模式
- 评估组件的职责分配

### 2. 开发架构分析
- 分析代码组织结构
- 识别技术栈和框架选择
- 评估开发工具链
- 分析代码质量和规范

### 3. 部署架构分析
- 分析部署环境和基础设施
- 识别容器化和编排策略
- 评估云服务和资源管理
- 分析部署流程和自动化

### 4. 运行架构分析
- 分析运行时组件交互
- 识别进程模型和线程管理
- 评估性能和资源使用
- 分析监控和日志策略

### 5. 数据架构分析
- 分析数据存储策略
- 识别数据模型和关系
- 评估数据流和处理
- 分析数据安全和备份

## 质量检查
- [ ] 所有五个架构视图都已分析
- [ ] 每个视图都有详细的技术评估
- [ ] 识别了潜在的风险和改进点
- [ ] 提供了具体的改进建议
